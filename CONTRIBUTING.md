# Contributing to Silogic

Thanks for your interest! Silogic is a compact, hackable PyTorch library for
**differentiable logic-gate networks** — faithful reimplementations of the core
papers (LILogic Net, convolutional logic-gate networks, WARP) plus extensions.
Contributions of all sizes are welcome: bug fixes, new node types, kernels,
examples, docs, and benchmarks.

By contributing you agree that your contributions are licensed under the
project's [MIT License](LICENSE).

## Getting started

```bash
git clone https://github.com/gaspardol/silogic
cd silogic
python -m venv .venv && source .venv/bin/activate      # or conda
pip install -e ".[dev]"          # core + pytest
pip install -e ".[triton]"       # optional: fused CUDA kernels (GPU only)
pip install -e ".[docs]"         # optional: to build the docs
```

Requires Python ≥ 3.9 and PyTorch ≥ 2.1. A CUDA GPU is optional — the Triton
kernels fall back to pure PyTorch, and the GPU tests auto-skip on CPU.

## Running the checks

```bash
pytest                                                 # test suite (CPU-friendly)
sphinx-build -b html docs docs/_build/html             # build the docs
```

CI runs `pytest` on Python 3.10–3.12 and builds the docs on every PR. Please make
sure both pass locally first.

## Project layout

| Path | What |
|---|---|
| `silogic/` | the installable package (layers, networks, data, training, Triton kernels) |
| `tests/` | pytest suite |
| `examples/` | runnable training scripts |
| `docs/` | Sphinx sources (user guide + autogenerated API reference) |

The `experiments/` folder (research scripts) and `data/`, `cache/`, `ckpt/` are
git-ignored local-only artifacts.

## Conventions

- **Every logic module implements both `forward` (the soft, differentiable
  circuit) and `forward_hard` (the discretized Boolean circuit).** New layers
  must provide both, and a test that they agree when the gates/connections are
  hardened. `forward_hard` must accept `uint8` *and* float `{0,1}` inputs.
- **Docstrings are Google-style** (`Args:` / `Returns:`), rendered into the API
  reference by Sphinx + napoleon. Document every argument, its valid
  values/choices, and its default.
- **Match the surrounding code style** — it is deliberately compact. Keep diffs
  focused; don't reformat unrelated code.
- **Add tests** for new behavior (`tests/`), and export new public objects from
  `silogic/__init__.py` *and* list them in `docs/api.rst`.

### Adding a new layer or network

1. Subclass `torch.nn.Module`; implement `forward` and `forward_hard`.
2. Export it in `silogic/__init__.py` (`__all__`).
3. Add it to the relevant `autosummary` block in `docs/api.rst`.
4. Add tests (shape, range, gradient, hardened soft==hard) — see existing
   `tests/` for patterns.

## Pull requests

1. Fork and create a topic branch.
2. Make your change with tests and docstrings; run `pytest` and the docs build.
3. Open a PR describing the change and linking any related issue. Keep PRs small
   and focused where possible.
